# How-To Guides

Operational procedures for using the Mechanistic Validity framework.

| Guide | What it does |
|---|---|
| [A — Audit a claim](A_audit-a-claim.md) | The 5-pass procedure for evaluating any MI claim |
| [B — Write a verdict](B_write-a-verdict.md) | The full verdict-statement template with worked examples |
| [C — Pick a mode tag](C_pick-a-mode-tag.md) | Decision tree for choosing the correct description-level tag |
| [D — Choose a metric](D_choose-a-metric.md) | Criterion → metric lookup table |
| [E — Report a baseline](E_report-a-baseline.md) | Baseline reporting rules (random-vector, untrained-model, published SOTA) |
| [F — Run a convergence check](F_run-a-convergence-check.md) | Multi-metric agreement protocol |
| [G — Handle disagreement](G_handle-disagreement.md) | What to do when metrics diverge |

## Order of operations for a new claim

1. **[C](C_pick-a-mode-tag.md)** — decide what level of claim you are making.
2. **[D](D_choose-a-metric.md)** — identify which metrics you need.
3. **[E](E_report-a-baseline.md)** — compute and report the required baselines.
4. **[F](F_run-a-convergence-check.md)** — check whether your metrics agree.
5. **If metrics disagree → [G](G_handle-disagreement.md)** before writing the verdict.
6. **[B](B_write-a-verdict.md)** — write the verdict statement.
7. **[A](A_audit-a-claim.md)** — audit your own claim before publishing.
